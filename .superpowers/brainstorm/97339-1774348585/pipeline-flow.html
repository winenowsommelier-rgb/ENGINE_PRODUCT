Description Import Pipeline — 3 Approaches

How should the CSV upload + AI processing flow work? Pick the approach that feels right.

A

Inline Tab on Products Page

Add an "Import Descriptions" tab directly on the existing Products page. Upload CSV, preview SKU matches, trigger AI processing — all without leaving the product browser.

Flow: Products page → Import tab → Upload CSV → Match preview → Process ✓

Pros

  • Stays in context of product list
  • Quick to access

Cons

  • Cramped for a multi-step workflow
  • Mixes browsing + bulk operations
B

Dedicated Description Manager Page ✦ Recommended

New page in the sidebar: Description Manager. A full 4-step wizard: Upload → Match & Preview → AI Rewrite & Validate → Publish to Products.

Flow:
1. Upload CSV → 2. Match SKUs → 3. AI Process → 4. Review & Publish

Pros

  • Clean space for multi-step workflow
  • New SKUs handled naturally in step 2
  • Full review before committing changes

Cons

  • Extra nav item in sidebar
C

Extend Existing Import Page

Add a "Description Mode" toggle to the current Import page. Reuse the existing batch pipeline — new CSV column mapping for EN/TH descriptions, feeds into the same processing queue.

Flow: Import page → Toggle "Description Mode" → Upload CSV → Batch process (same pipeline)

Pros

  • Reuses existing infrastructure
  • Fewer new files/pages

Cons

  • Import page already complex
  • AI rewrite step doesn't fit current model
  • Less visibility into per-product status
