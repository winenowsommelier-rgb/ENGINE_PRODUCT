Run Workflow UI — What to Build Next

Spec: docs/superpowers/specs/2026-05-29-supplier-intake-run-workflow-design.md — reviewed, approved, committed.

User Flow

1. Pick supplier
2. Browse Drive
3. Start Run
4. Auto-chain
5. Review
6. Commit
  • Phase setup: Supplier dropdown → Drive file list → "Start Run"
  • Phase processing: Auto-chain normalize → match → price with stepper bar
  • Phase review: Table with checkboxes, Export CSV, Adjust Settings link
  • Phase committed: Summary card + "Start another run"

Review Table Columns

  • ☐ checkbox (pre-checked if price.status !== 'needs_review')
  • # row_number · Name · Matched SKU
  • Confidence (green≥100, amber 55-99, red<55)
  • Cost · Supplier RSP · Calculated Price · Margin%
  • Status badge: auto ✓ review ⚑ flagged ✗
  • Issues (collapsed, expand on hover)

Key Rules

  • Flagged/blocked rows → no checkbox, greyed, never committed
  • needs_review rows → unchecked by default
  • Export CSV: text/csv, all 13 columns, issues joined with |
  • "Adjust Settings" → calls onNavigateToSettings() prop

Supplier Code Conventions

Code Structure

  • Root code (e.g. AB) — main commercial relationship with distributor
  • Variant number (e.g. AB2, AB3) — separate supplier in same family
  • Variant bracket (e.g. AB(2), FX(2)) — consignment sub-line of same supplier
  • Distributor sub-line (e.g. AD2, AD3) — same distributor (IWS), different brand owner product line

Key Distributor Relationships

  • AD / AD2 / AD3 = IWS (distributor). AD=IWS direct, AD2=IWS(MH Wine), AD3=IWS(TWE/Penfolds). Brand owners are reference only — pricing & intake via IWS.
  • BU / BU(2) / BU(4) / BU(8) / BU(9) / BU(11) = Suriwongse Store / related retail suppliers. All share the Surawong Store Drive folder.
  • AA / AA2 / AA4 = Italasia (same folder, different product categories)
  • HM / HP — HM=JDSS, HP=Vinobox (HP was reassigned from JDSS duplicate)
